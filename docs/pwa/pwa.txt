1. The "Wake Up" Logic

The best event to catch the exact moment the app comes back into focus is visibilitychange. To ensure it doesn't mess with your normal webpage behavior, you can wrap your logic in a check to see if the app is running in "standalone" mode (as a PWA).
JavaScript

document.addEventListener("visibilitychange", () => {
  // Check if the page is now visible
  if (document.visibilityState === 'visible') {
    
    // Optional: Only run special logic if launched as a PWA
    const isPWA = window.matchMedia('(display-mode: standalone)').matches;

    if (isPWA) {
       console.log("PWA Resumed. Checking for stale data...");
       refreshAppState();
    }
  }
});

2. Why your Status is "Wrong"

When a mobile OS (Android/iOS) puts a PWA in the background, it often "freezes" the JavaScript execution to save battery.

    The Problem: If you have a setInterval running, it stops. When you open the app 3 hours later, your app might think only 1 second has passed because the clock "paused" for it.

    The Fix: Always calculate state based on the System Clock (Date.now()), not on a counter or interval.

3. Using the pageshow Event

If the browser actually kills the process to save memory (a "Cold Start"), visibilitychange might not fire the way you expect. In this case, pageshow is a more robust alternative for catching the "back-to-life" moment.
JavaScript

window.addEventListener('pageshow', (event) => {
  if (event.persisted) {
    // The page was restored from the "back/forward" cache (BFCache)
    // This is a common way PWAs "wake up" with old data
    location.reload(); // Or call your refresh function
  }
});

Summary Table: Which Event to Use?
Scenario	Best Event	Why?
Switching back from another app	visibilitychange	Fires every time the tab/window becomes the primary view.
Phone woke up from sleep	visibilitychange	Usually triggers as soon as the screen turns on and Chrome is active.
App was "frozen" in memory	pageshow	Specifically detects if the page is being loaded from a cached state.
User clicked a Notification	notificationclick	(Service Worker) Use this if the focus comes from a push notice.
Pro-Tip: The "Stale Check"

To avoid unnecessary reloading, store a timestamp when the app goes to the background. When it returns, check if more than (for example) 30 minutes have passed. If yes, force a data refresh; if no, let the user continue where they left off.
